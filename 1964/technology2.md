# ARPANET 的誕生

## 一、ARPANET 的起源

### 1.1 冷戰背景

1964 年，冷戰正酣。美國國防部希望建立一個能夠在核戰爭中存活下來的通信網路。傳統的中央控制式網路在節點被摧毀後可能會癱瘓。

### 1.2 分組交換理論

1961-1962 年，保羅·巴蘭（Paul Baran）提出了一種新的網路概念——分組交換（Packet Switching）。這種網路沒有中央控制，數據被分割成小包（packets），每個包可以獨立選擇路由傳輸。

### 1.3 ARPA 的角色

美國國防部高級研究計畫局（ARPA）決定資助這項研究。1964 年，ARPANET 的設計工作正式啟動。

## 二、網路設計原則

### 2.1 關鍵設計目標

ARPANET 的設計者確定了以下目標：
- **可靠性**：部分網路故障不應影響整體通信
- **異構性**：連接不同類型的電腦和網路
- **簡單性**：網路只負責數據傳遞，應用功能由端設備實現

### 2.2 開放架構

ARPANET 採用開放架構，任何人都可以連接。這與當時其他專有的網路系統不同。

## 三、技術架構

### 3.1 分組交換

數據被分割成固定大小的分組（packets），每個分組包含：
- 目標地址
- 源地址
- 序列號
- 數據

### 3.2 路由選擇

網路中的每個節點（稱為 IMP，Interface Message Processor）獨立地決定每個分組的下一跳。

### 3.3 流量控制

採用「存儲-轉發」機制，每個 IMP 會緩存收到的分組，然後轉發到下一個節點。

## 四、節點部署

### 4.1 首批節點

ARPANET 最初連接了四個節點：
1. UCLA（加州大學洛杉矶分校）
2. SRI（斯坦福研究院）
3. UCSB（加州大學聖塔芭芭拉分校）
4. 猶他大學

### 4.2 擴展

1969 年 ARPANET 正式開通時，已經有四個節點。到 1970 年代初，節點數量增加到數十個。

## 五、應用和協議

### 5.1 早期應用

ARPANET 的首批應用包括：
- 遠程登錄（Telnet）
- 文件傳輸（FTP）
- 電子郵件

### 5.2 網路協議

1970 年代初，ARPANET 開發了網路控制程序（NCP）等協議。這些協議在 1974 年被 TCP/IP 取代。

## 六、對後世的影響

ARPANET 的設計和運行經驗對後世網路技術产生了深远影响：

1. **互聯網**：ARPANET 是今天互聯網的直接祖先
2. **TCP/IP**：ARPANET 採用的 TCP/IP 協定成為互聯網的標準
3. **網路文化**：ARPANET 開創了網路文化和協作方式
4. **商業化**：ARPANET 的商業化催生了電信產業
