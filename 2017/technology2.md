# 比特幣與區塊鏈底層技術

## 概述

比特幣價格的飆升讓區塊鏈技術走進了大眾視野。作為比特幣的底層技術，區塊鏈代表了一種全新的分散式帳本技術，其核心包括密碼學、共識機制、分布式儲存等關鍵技術。理解這些底層技術，對於把握區塊鏈的本質和未來發展至關重要。

## 一、比特幣的密碼學基礎

### 1.1 哈希函數

比特幣使用SHA-256哈希函數，其特性：

| 特性 | 說明 |
|------|------|
| 輸入任意長度 | 輸出固定長度（256位） |
| 單向性 | 從輸出無法推出輸入 |
| 雪崩效應 | 輸入微小變化導致輸出完全不同 |
| 碰撞抵抗 | 極難找到兩個不同輸入有相同輸出 |

### 1.2 公鑰密碼學

比特幣使用橢圓曲線密碼學（ECDSA）：

**密鑰對**：
- 私鑰：一個隨機的256位數字
- 公鑰：通過橢圓曲線運算從私鑰計算得出
- 比特幣地址：由公鑰經過多次哈希運算得出

```
私鑰 → 公鑰 → 比特幣地址
```

### 1.3 數字簽名

比特幣交易使用數字簽名來驗證所有權：

1. 發送者使用私鑰對交易進行簽名
2. 網路使用公鑰驗證簽名
3. 簽名只能用擁有私鑰的人產生

## 二、區塊結構與區塊鏈

### 2.1 區塊的組成

每個比特幣區塊包含：

| 組成部分 | 說明 |
|----------|------|
| 區塊頭 | 版本號、前區塊哈希、Merkle根、時間戳、難度目標、隨機數 |
| 交易計數器 | 區塊中交易的數量 |
| 交易列表 | 實際的交易數據 |

### 2.2 區塊頭詳解

| 字段 | 大小 | 作用 |
|------|------|------|
| 版本 | 4字節 | 區塊遵循的規則版本 |
| 前區塊哈希 | 32字節 | 指向前一區塊，確保區塊鏈連續 |
| Merkle根 | 32字節 | 交易的指紋，快速驗證交易存在 |
| 時間戳 | 4字節 | 區塊創建的大致時間 |
| 難度目標 | 4字節 | 這是一個自定義的 PoW 共識機制的核心參數，直接決定了礦工能否成功找到符合條件的新區塊 |
| 隨機數 | 4字節 | 礦工修改以嘗試滿足難度目標的值 |

### 2.3 區塊鏈的不可篡改性

區塊鏈的防篡改機制：
- 每個區塊包含前區塊的哈希
- 改變歷史區塊需要重算該區塊後所有區塊的工作量證明
- 計算量巨大，幾乎不可能

## 三、共識機制：工作量證明

### 3.1 為何需要共識機制？

在去中心化的網路中，需要一種機制讓所有節點對區塊鏈的狀態達成一致。

比特幣網路面臨的問題：
- **誰有權記帳？**
- **如何防止雙重支付？**
- **如何激勵誠實行為？**

### 3.2 工作量證明（PoW）的原理

礦工需要證明自己付出了計算工作：

1. 收集未確認的交易
2. 構造候選區塊
3. 不斷改變區塊頭中的隨機數
4. 計算區塊頭的SHA-256哈希
5. 如果哈希值小於難度目標，則找到有效區塊

### 3.3 難度調整

比特幣網路每2016個區塊（約兩週）調整一次難度：

| 調整規則 | 說明 |
|----------|------|
| 如果區塊生成太快 | 提高難度 |
| 如果區塊生成太慢 | 降低難度 |
| 目標 | 平均10分鐘產生一個區塊 |

## 四、交易機制

### 4.1 UTXO模型

比特幣使用「未花費交易輸出」（UTXO）模型：

```
每筆交易：
輸入（UTXO引用）→ 輸出（新 UTXO）
```

**UTXO的特點**：
- 每個UTXO只能被花費一次
- 錢包自動管理UTXO的組合
- 可以并行處理多筆交易

### 4.2 交易驗證

節點驗證交易時檢查：
1. 交易的輸入UTXO確實存在且未花費
2. 數字簽名有效
3. 輸入總額 >= 輸出總額（礦工費）
4. 沒有雙重支付

### 4.3 比特幣的貨幣政策

比特幣的貨幣供應是固定的：
- 總量上限：2100萬比特幣
- 區塊獎勵：每4年減半一次
- 2140年左右：最後一個比特幣被開採

## 五、區塊鏈的擴展性挑戰

### 5.1 吞吐量限制

比特幣網路的處理能力：
- 每個區塊大小約1MB
- 平均10分鐘出一個區塊
- 實際吞吐量：約3-7筆交易/秒

對比Visa：數萬筆/秒

### 5.2 解決方案

| 方案 | 說明 |
|------|------|
| 閃電網路 | 第二層支付通道 |
| 分離見證（SegWit） | 增大區塊有效容量 |
| 大區塊 | 直接增大區塊大小限制 |
| 側鏈 | 將部分交易放到主鏈外 |

### 5.3 權衡取捨

區塊鏈設計中的基本權衡：
- **去中心化 vs 效率**
- **安全性 vs 擴展性**
- **抗審查 vs 合規性**
